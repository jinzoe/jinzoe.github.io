A Self-Attentive model for Knowledge Tracing
AIED 2019

Abstract

近年来，基于递归神经网络（RNN）的深度知识追踪（Deep Knowledge Tracing，DKT）和动态键值记忆网络（Dynamic Key-Value Memory Network，DKVMN）由于能够捕捉复杂的人类学习表征，其性能优于传统方法。然而，这些方法在处理稀疏数据时面临着不能很好地进行泛化的问题，而真实世界的数据就是这样，因为学生与很少的KCs交互。

为了解决这个问题，我们开发了一种方法，从学生过去的活动中识别出与给定的KC相关的KCs，并基于所选择的相对较少的KCs预测他/她的掌握程度。由于预测是基于相对较少的过去的活动，它处理数据稀疏问题比基于RNN的方法更好。为了确定KCs之间的相关性，我们提出了一种基于自注意力机制的方法——自注意力知识追踪（SAKT）。在各种真实数据集上的大量实验表明，我们的模型在知识跟踪方面的性能优于最先进的模型，平均提高了4.43%的AUC。

Introduction

在本文中，我们提出了一种基于纯注意机制的方法——transformer。在KT任务中，学生在进行一系列学习活动时所建立的技能是相互关联的，而他在某一特定练习中的表现取决于他在过去与该练习相关的练习中的表现。例如，在图1中，一个学生来解决一个题目对“二次方程”（题5）属于知识概念“方程”，他需要知道如何找到“开根方”（题3）和“线性方程”（题4）。

SAKT，本文首次提出根据过去的交互识别相关的KC，然后预测学生的表现。我们使用练习题作为KCs。正如我们稍后所展示的，SAKT在预测学生在特定练习中的表现时，为之前回答的练习分配权重。所提出的SAKT方法显著优于最先进的KT方法，在AUC上的性能平均提高了4.43%。此外，SAKT的主成分（自我注意）适合并行性；因此，使我们的模型数量级比基于RNN的模型更快。

嵌入层

对得到的输入序列y=（y1，y2，…，yt）转换为s=（s1，s2，.…，sn)。其中n为模型能够处理的最大长度。由于该模型可以处理固定长度序列的输入，如果序列长度tn，我们将序列划分为长度为n的子序列。所有这些子序列都作为模型的输入。

我们训练一个交互嵌入矩阵，其中d为潜在维数。该矩阵用于获取序列中每个元素的嵌入量（Msi）。同样，我们训练练习嵌入矩阵使得集合ei中的每个练习都嵌入到eith中行。

位置编码：位置编码是在自注意神经网络中对位置进行编码的一层，与卷积网络和递归神经网络一样，我们可以对序列的顺序进行编码。这一层在知识追踪问题中尤为重要，因为学生的知识状态是随着时间的推移而逐步稳定发展的。特定时间的知识状态不应该显示波形转换。为了将其结合起来，我们使用了一个参数，位置嵌入，这是在训练中学习的。将位置嵌入矩阵的第i行Pi加入到交互序列的第i个元素的交互嵌入向量中。

前馈层

上面描述的自我注意层会产生加权的值和，即之前交互作用的Vi。然而矩阵的行从多线程层，获得仍然是一个值的线性组合。为了在模型中考虑非线性和不同潜在维数之间的相互作用，我们使用了一个前馈网络。

残差连接

使用剩余连接将低层特征传播到高层。因此，如果低层特征对于预测很重要，那么剩余连接将有助于将它们传播到执行预测的最终层。在KT的背景下，学生尝试属于的练习加强这一概念的具体概念。因此，剩余连接可以帮助将最近解决的练习的嵌入传播到最后一层，使模型更容易利用低层信息。在自注意和前馈层之后应用剩余连接。

层归一化

跨特征的归一化输入有助于神经网络的稳定和加速。出于同样的目的，我们在体系结构中使用了层规范化。层归一化同时应用于自注意层和前馈层。

Result

学生成绩预测

表3为SAKT与当前最先进方法的成绩对比。在综合数据集上，SAKT的性能优于其他竞争方法，AUC为0.832，而DKT+为0.824。尽管Synthetic是最密集的数据集，但由于生成Synthetic的方法，SAKT的性能优于基于RNN的方法。对于这个数据集，每个单独的练习只派生自一个概念。学生回答一个练习的概率从这个数据集确定正确使用项目反应理论

c表示猜测正确的概率，a和β是随机选择的数字来表示概念的能力和运动困难，分别。因此，在这个数据集中，属于同一概念的练习具有很强的相关性。

与其他基准测试不同，SAKT直接尝试识别属于相同概念的练习，因此性能优于其他方法。在ASSIST2009上，SAKT的性能优于其他方法，比性能第二好的方法提高了3.16%。对于ASSIST2015数据集，SAKT显示了令人印象深刻的15.87%的改进。我们将这一优势归因于SAKT利用的注意力机制，即使数据集是稀疏的，也可以很好地学习和推广，而在ASSIST2015中，它的密度是其他数据集中最小的。对于STATICS2011，我们的方法与DKT+相比，性能提高了2.16%。对于ASSISTChall，我们的方法的性能与DKT相当。这可以归因于ASSISTChall是所有真实数据集中最密集的数据集。

注意力权重可视化

将过去交互的元素（充当键）和学生接下来要解决的练习（充当查询）之间的注意力权重可视化，有助于理解过去交互中的哪些练习与查询练习相关。有了这个动机，我们计算了所有序列中每个练习对（e1，e2）的注意权重之和，其中e1用作查询，而与练习e2的交互用作键。然后，我们将注意力权重标准化，使每个查询的权重之和为1。这就产生了一个关联矩阵，其中每个元素（e1，e2）表示e2对e1的影响。我们对合成数据进行分析，这个数据集是用已知的隐藏概念生成的。图3a为与合成题相关矩阵对应的热图。对于合成，所有序列都由相同序列中的所有练习标记组成，从1到50。

为了构建练习标签之间的影响图，如图3b所示，我们使用了关联矩阵。首先，我们画出了第一序列中的运动属于每一个隐藏的概念，并访问关联矩阵的每一行，并连接相对应的运动这一行第一个两个练习排名基于边缘的体重，这之间注意重量成正比的练习。我们可以看到，基于注意力权重，我们能够实现完美的聚类运动标签的基础上，隐藏的概念，他们的来源。一个有趣的观察是，两个练习在序列上相隔很远，但属于同一个概念，可以被SAKT识别。例如，如图3b所示，一个关于练习22的查询将大部分权重分配给了练习5中的键，即使它们在序列中相隔很远。

两种相互关联的练习往往具有较高的注意力权重，因为其中一种练习的表现会影响另一种练习的表现。此外，在实际场景中，顺序相近的练习往往属于同一个概念。因此，我们期望注意力的权重偏向于最近在交互序列中发生的练习。为了说明这一点，我们手动分析了ASSIST2009数据集，以可视化所选样本的注意权重。表4显示了一些练习，以及过去的交互和分配给每个交互的注意力权重。
